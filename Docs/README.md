# Docs/

Long-form documentation for the Zonit stack — concepts, architecture decision
records, migration guides, and design notes that don't fit naturally inside a
runnable demo page.

Each top-level Markdown file is one self-contained topic. Use the runnable
demos under `Source/Example/` for tutorial-style content (anything you'd want
to *show*, not just describe) and keep this folder for the things that need to
be *read*.

## Planned structure

```
Docs/
├── README.md                      (this file)
├── architecture/                  high-level concepts: Site / Area / Layout / Theme
├── adr/                           Architecture Decision Records (numbered)
├── packages/                      one file per published Zonit.* package
└── guides/                        end-to-end how-tos (auth, multi-tenant, ...)
```
